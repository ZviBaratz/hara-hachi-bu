Notes for EGO reviewers — Hara Hachi Bu
========================================

pkexec/polkit usage
-------------------
This extension writes battery charging thresholds to sysfs paths
(/sys/class/power_supply/BAT*/charge_control_{start,end}_threshold) which
require root privileges. A helper script (hhb-power-ctl) is executed via
pkexec with:
- Strict input validation (integers 0-100, whitelisted sysfs filenames)
- Path traversal prevention (realpath -e with /sys/ prefix check)
- set -eu for fail-fast behavior
- Polkit rules restrict to active local sessions in the sudo/wheel group

Private API usage
-----------------
lib/uiPatcher.js accesses QuickSettingsMenu._indicators to manage the
built-in power profile indicator visibility. No public API exists for this.
Tested on GNOME 46-50. Fallback strategies handle API changes gracefully.

Clipboard usage
---------------
Extension copies the helper installation command to clipboard when the user
clicks "Copy" in the setup prompt. Uses St.Clipboard with visual
confirmation feedback.

MockDevice
----------
lib/device/MockDevice.js is excluded from the distribution package. It
exists in the source repository for developer testing only.

Scheduled profiles
------------------
Profiles support optional time-based schedules (per-day, start/end time).
GLib.timeout_add_seconds drives a schedule timer capped at 3600s to guard
against clock drift. Overnight schedules (start > end) are supported.
Suspend/resume detection uses org.freedesktop.login1.Manager PrepareForSleep
D-Bus signal to re-evaluate schedules after waking from sleep.

Boost charge
------------
One-click temporary charge to 100% via a state machine with auto-revert
triggers: battery full, configurable safety timeout (1-12h), and AC
disconnect. Temporarily overrides thresholds and pauses auto-management,
restoring the previous state on deactivation.

Multi-battery support
---------------------
The helper script (hhb-power-ctl) supports BAT0 through BAT3 for systems
with multiple batteries. DeviceManager auto-detects available batteries by
enumerating /sys/class/power_supply and falls back to BAT0-BAT3 if
enumeration is unavailable.

Force discharge
---------------
Controls the charge_behaviour sysfs file to enable force discharge on
supported hardware (primarily ThinkPad). All writes go through the helper
script with validated mode values. A file monitor tracks external changes.

Logging convention (EGO-A-004)
------------------------------
A DEBUG flag in lib/helper.js gates all diagnostic logging through
debugLog/debugWarn/debugError helpers that are no-ops when DEBUG is false.
Only user-actionable failures (D-Bus proxy init errors, missing helper
prompts) use ungated console.error calls. This keeps per-file ungated
console.* call counts well under the EGO-A-004 threshold.

Async patterns
--------------
Controllers use a _destroyed flag checked after every await point to prevent
callbacks from executing on destroyed objects during extension reload.
Gio._promisify wraps D-Bus proxy creation. Promise chains guard all
continuations with _destroyed checks. These are standard GJS async safety
patterns for extension lifecycle management.
